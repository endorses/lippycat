# Glossary

This appendix defines key terms used throughout the lippycat user manual.

| Term | Definition |
|------|-----------|
| **ADMF** | Administration Function. The external system that manages Lawful Interception tasks by sending activation/deactivation requests over the X1 interface. See [Lawful Interception](../part5-advanced/lawful-interception.md). |
| **AF_XDP** | Address Family XDP. A Linux kernel bypass mechanism that uses eBPF/XDP to deliver packets directly to userspace, enabling high-speed capture without the overhead of the traditional socket path. See [Performance Optimization](../part5-advanced/performance.md). |
| **Aho-Corasick** | A string matching algorithm that searches for multiple patterns simultaneously in a single pass. Used by lippycat for efficient multi-pattern filtering (e.g., matching many SIP URIs at once). See [Performance Optimization](../part5-advanced/performance.md). |
| **Auto-Rotating PCAP** | A non-VoIP capture feature that automatically rotates PCAP output files based on size or time thresholds, preventing any single file from growing unbounded. See [Central Aggregation with `lc process`](../part3-distributed/process.md). |
| **BPF** | Berkeley Packet Filter. A kernel-level packet filtering language that allows capture tools to discard uninteresting packets before they reach userspace, reducing CPU and memory overhead. See [BPF Filter Reference](bpf-reference.md). |
| **Build Tag** | A Go compiler directive that controls which source files are included in a binary. lippycat uses build tags (`all`, `hunter`, `processor`, `cli`, `tui`, `li`, `cuda`) to produce specialized binary variants. See [Installation & Setup](../part1-foundations/installation.md). |
| **CA** | Certificate Authority. An entity that issues and signs TLS certificates, establishing a chain of trust. In lippycat's distributed architecture, a CA signs certificates for hunters, processors, and TUI clients. See [Security](../part5-advanced/security.md). |
| **Call-ID** | A globally unique SIP header value that identifies a call across all related SIP messages (INVITE, BYE, ACK, etc.) and associates RTP media streams with signaling. See [Protocol Deep Dives](../part5-advanced/protocol-deep-dives.md). |
| **Capture Interface** | The network interface (physical NIC, virtual interface, or loopback) from which lippycat reads packets. Specified with the `-i` / `--interface` flag. See [CLI Capture with `lc sniff`](../part2-local-capture/sniff.md). |
| **CC** | Communication Content. The actual packet payload (media, data) delivered over the X3 interface in Lawful Interception. See [Lawful Interception](../part5-advanced/lawful-interception.md). |
| **Circuit Breaker** | A resilience pattern that stops forwarding attempts after repeated failures, preventing resource exhaustion. Used in hierarchical processor-to-processor forwarding. See [Distributed Architecture Overview](../part3-distributed/architecture.md). |
| **Codec** | Coder-decoder. An algorithm that encodes and decodes audio or video for transmission over RTP. Common VoIP codecs include G.711 (uncompressed), G.729 (compressed), and Opus (adaptive). See [Protocol Deep Dives](../part5-advanced/protocol-deep-dives.md). |
| **Command Hook** | A shell command executed automatically when a PCAP file is closed or a VoIP call completes. Configured with `--pcap-command` and `--voip-command` flags. See [Central Aggregation with `lc process`](../part3-distributed/process.md). |
| **CUDA** | Compute Unified Device Architecture. NVIDIA's GPU computing platform, used by lippycat for hardware-accelerated packet filtering on hunter and tap nodes. Requires the `cuda` build tag. See [Performance Optimization](../part5-advanced/performance.md). |
| **Disk Overflow Buffer** | A hunter feature that writes captured packets to local disk when the connection to the processor is lost, then replays them upon reconnection. Prevents data loss during network outages. See [Edge Capture with `lc hunt`](../part3-distributed/hunt.md). |
| **DNS** | Domain Name System. A protocol that translates human-readable domain names into IP addresses. lippycat can capture and analyze DNS queries and responses. See [Protocol Deep Dives](../part5-advanced/protocol-deep-dives.md). |
| **ETSI** | European Telecommunications Standards Institute. The standards body that defines the X1/X2/X3 interfaces used for Lawful Interception. See [Lawful Interception](../part5-advanced/lawful-interception.md). |
| **Filter** | A named rule (e.g., `sip_user`, `dns_domain`, `ip_address`) that determines which packets a hunter captures and forwards. Filters are pushed from the processor to hunters over gRPC. See [Core Concepts](../part1-foundations/core-concepts.md). |
| **Flow Control** | The mechanism by which a processor signals hunters to adjust their sending rate. Uses four states: CONTINUE (normal), SLOW (reduce rate), PAUSE (stop sending), and RESUME (restart after pause). See [Distributed Architecture Overview](../part3-distributed/architecture.md). |
| **gRPC** | Google Remote Procedure Call. A high-performance RPC framework used for communication between hunters and processors in the distributed architecture. See [Distributed Architecture Overview](../part3-distributed/architecture.md). |
| **HTTP** | Hypertext Transfer Protocol. The foundation protocol of the web. lippycat can capture and analyze HTTP requests and responses. See [Protocol Deep Dives](../part5-advanced/protocol-deep-dives.md). |
| **Hub-and-Spoke** | A deployment topology where a single processor receives captured packets from multiple hunters. The simplest and most common distributed configuration. See [Distributed Architecture Overview](../part3-distributed/architecture.md). |
| **Hunter** | An edge capture node that sniffs packets on a network interface, applies filters, and forwards matching packets to a processor via gRPC. Hunters are lightweight and can be deployed across multiple network segments. See [Edge Capture with `lc hunt`](../part3-distributed/hunt.md). |
| **Hierarchical Mode** | A deployment topology where processors forward packets to upstream processors, creating a multi-tier aggregation hierarchy for large-scale deployments. See [Distributed Architecture Overview](../part3-distributed/architecture.md). |
| **IMAP** | Internet Message Access Protocol. A protocol for retrieving email from a mail server, allowing messages to remain on the server. See [Protocol Deep Dives](../part5-advanced/protocol-deep-dives.md). |
| **IRI** | Intercept Related Information. Signaling metadata (e.g., SIP headers, call setup details) delivered over the X2 interface in Lawful Interception. See [Lawful Interception](../part5-advanced/lawful-interception.md). |
| **JA3** | A TLS client fingerprinting method that generates a hash from Client Hello parameters (cipher suites, extensions, elliptic curves). Used to identify client applications regardless of IP address. See [Protocol Deep Dives](../part5-advanced/protocol-deep-dives.md). |
| **JA3S** | A TLS server fingerprinting method that generates a hash from Server Hello parameters. The server-side complement to JA3. See [Protocol Deep Dives](../part5-advanced/protocol-deep-dives.md). |
| **JA4** | A next-generation TLS fingerprinting method and successor to JA3, providing improved accuracy and additional transport context. See [Protocol Deep Dives](../part5-advanced/protocol-deep-dives.md). |
| **LI** | Lawful Interception. Authorized network monitoring conducted under legal authority, implemented via ETSI X1/X2/X3 interfaces. Requires the `li` build tag. See [Lawful Interception](../part5-advanced/lawful-interception.md). |
| **libpcap** | The standard C library for network packet capture on Unix-like systems. lippycat uses libpcap (via gopacket) for reading packets from interfaces and PCAP files. See [Core Concepts](../part1-foundations/core-concepts.md). |
| **MDF** | Mediation/Delivery Function. The law enforcement facility that receives intercepted data (IRI over X2, CC over X3) from the network element. See [Lawful Interception](../part5-advanced/lawful-interception.md). |
| **mDNS** | Multicast DNS. A protocol (port 5353) that resolves hostnames to IP addresses on local networks without a central DNS server. See [Protocol Deep Dives](../part5-advanced/protocol-deep-dives.md). |
| **MOS** | Mean Opinion Score. A numerical voice quality metric ranging from 1.0 (unintelligible) to 5.0 (excellent). Estimated from RTP stream statistics such as jitter and packet loss. See [Protocol Deep Dives](../part5-advanced/protocol-deep-dives.md). |
| **mTLS** | Mutual TLS. A TLS configuration where both the client and server present certificates for authentication, providing bidirectional identity verification. Required for production distributed deployments. See [Security](../part5-advanced/security.md). |
| **NE** | Network Element. In ETSI LI terminology, the system performing the interception -- in this case, lippycat itself (typically the processor node). See [Lawful Interception](../part5-advanced/lawful-interception.md). |
| **OpenCL** | Open Computing Language. An open standard for parallel computing on GPUs, CPUs, and other accelerators. An alternative GPU backend to CUDA for packet filtering. See [Performance Optimization](../part5-advanced/performance.md). |
| **P-Asserted-Identity** | A SIP header that carries the verified identity of the calling party, as asserted by a trusted intermediary. Used for caller identification in VoIP analysis. See [Protocol Deep Dives](../part5-advanced/protocol-deep-dives.md). |
| **PCAP** | Packet Capture. A standard file format (`.pcap`) for storing captured network packets, compatible with tools such as Wireshark and tcpdump. See [Core Concepts](../part1-foundations/core-concepts.md). |
| **Per-Call PCAP** | A VoIP-specific feature that writes a separate PCAP file for each SIP call, containing all signaling and media packets associated with that call's Call-ID. See [Central Aggregation with `lc process`](../part3-distributed/process.md). |
| **POP3** | Post Office Protocol version 3. A protocol for retrieving email from a mail server, typically downloading and removing messages from the server. See [Protocol Deep Dives](../part5-advanced/protocol-deep-dives.md). |
| **Processor** | A central aggregation node that receives packets from one or more hunters via gRPC, performs protocol analysis, writes PCAP files, and serves the TUI interface. See [Central Aggregation with `lc process`](../part3-distributed/process.md). |
| **Production Mode** | An operational mode activated by setting `LIPPYCAT_PRODUCTION=true`, which enforces TLS encryption on all connections and blocks the `--insecure` flag. See [Security](../part5-advanced/security.md). |
| **Promiscuous Mode** | A network interface mode that captures all traffic on the network segment, not just traffic addressed to the host. Required for monitoring traffic between other hosts. See [CLI Capture with `lc sniff`](../part2-local-capture/sniff.md). |
| **RTP** | Real-time Transport Protocol. A protocol that carries VoIP audio and video streams, providing sequencing, timestamping, and payload type identification. See [Protocol Deep Dives](../part5-advanced/protocol-deep-dives.md). |
| **SDP** | Session Description Protocol. A format used within SIP signaling to negotiate media session parameters such as codecs, IP addresses, and ports. See [Protocol Deep Dives](../part5-advanced/protocol-deep-dives.md). |
| **SIMD** | Single Instruction, Multiple Data. CPU vector extensions (AVX2, SSE4.2) that process multiple data elements in parallel, used by lippycat for accelerated pattern matching. See [Performance Optimization](../part5-advanced/performance.md). |
| **SIP** | Session Initiation Protocol. A signaling protocol used to establish, modify, and terminate VoIP calls. lippycat analyzes SIP messages for call tracking and filtering. See [Protocol Deep Dives](../part5-advanced/protocol-deep-dives.md). |
| **SMTP** | Simple Mail Transfer Protocol. The standard protocol for sending email between mail servers. See [Protocol Deep Dives](../part5-advanced/protocol-deep-dives.md). |
| **SNI** | Server Name Indication. A TLS extension in the Client Hello message that specifies the hostname the client is connecting to, enabling virtual hosting over TLS. See [Protocol Deep Dives](../part5-advanced/protocol-deep-dives.md). |
| **SRTP** | Secure RTP. An encrypted variant of RTP that provides confidentiality, authentication, and replay protection for VoIP media streams. See [Protocol Deep Dives](../part5-advanced/protocol-deep-dives.md). |
| **Tap** | A standalone node that combines hunter and processor capabilities without gRPC, running capture and analysis locally in a single process. See [Standalone Mode with `lc tap`](../part3-distributed/tap.md). |
| **TCP Performance Mode** | Configuration profiles that control TCP reassembly behavior, trading off between completeness and speed. Modes include `balanced`, `high_performance`, `low_latency`, and `minimal`. See [Performance Optimization](../part5-advanced/performance.md). |
| **TLS** | Transport Layer Security. A cryptographic protocol that provides encrypted communication between networked applications. lippycat both analyzes TLS traffic and uses TLS to secure its own gRPC connections. See [Security](../part5-advanced/security.md). |
| **TUI** | Terminal User Interface. lippycat's interactive terminal-based monitoring interface, built with the Bubbletea framework. Provides real-time packet display, hunter management, and call monitoring. See [Interactive Capture with `lc watch`](../part2-local-capture/watch-local.md). |
| **Virtual Interface (VIF)** | A software-created network interface (e.g., tap/tun device) used for packet injection, replay, or inter-VM communication. Can be used as a capture interface. See [CLI Capture with `lc sniff`](../part2-local-capture/sniff.md). |
| **X1** | The ETSI Lawful Interception administration interface. Carries task activation and deactivation requests between the ADMF and the network element over HTTPS/XML. See [Lawful Interception](../part5-advanced/lawful-interception.md). |
| **X2** | The ETSI Lawful Interception IRI delivery interface. Carries signaling metadata (Intercept Related Information) from the network element to the MDF over TLS. See [Lawful Interception](../part5-advanced/lawful-interception.md). |
| **X3** | The ETSI Lawful Interception CC delivery interface. Carries communication content (actual packet data) from the network element to the MDF over TLS. See [Lawful Interception](../part5-advanced/lawful-interception.md). |
